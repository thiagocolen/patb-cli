# Documentation Content Plan

This document plans the real content for the `patb-cli` documentation website scaffolded
in `docs/` (see `docs/planning/docusaurus_plan.md` for the original scaffolding plan).
It replaces the default `create-docusaurus` placeholder content with material derived
from `src/` and `README.md`, organized into an End-User section and a Developer &
Technical section.

---

### 1. Source Material

- `README.md`: features, architecture diagram, installation/setup, usage (REPL and Zed
  ACP bridge), CLI flags, examples, troubleshooting.
- `src/index.ts`: entrypoint, `--bridge`/`-b` and `--help`/`-h` flag handling.
- `src/config.ts`: `.env` / environment variable loading (`PATBA_API_KEY`), `maskKey`.
- `src/cli.ts`: interactive REPL loop (readline-based), progress/response rendering.
- `src/bridge.ts`: Zed ACP JSON-RPC 2.0 bridge (`initialize`, `session/new`,
  `session/prompt`, `agents/list`, error codes).
- `src/remote.ts`: HTTPS client for the remote agent service
  (`d33ib4uu7f4xpi.cloudfront.net`) — thread creation, run triggering, SSE stream parsing.

All facts used in the new docs come directly from these files — no invented behavior.

---

### 2. Information Architecture

The Docusaurus sidebar is auto-generated from the `docs/docs/` folder tree
(`sidebars.ts` → `{type: 'autogenerated', dirName: '.'}`), so the IA below is expressed
purely as a folder structure using `_category_.json` and `sidebar_position` frontmatter.

```
docs/docs/
├── intro.mdx                              (replaces default intro, position 1)
├── end-user/                              ("End-User Documentation", position 2)
│   ├── installation.mdx                   (prereqs, npm install, npm run build, npm link)
│   ├── configuration.mdx                  (.env, PATBA_API_KEY, env var option, masking)
│   ├── cli-usage.mdx                      (REPL walkthrough, controls, exit, transcript)
│   ├── zed-bridge-usage.mdx               (bridge mode, settings.json snippet)
│   ├── command-reference.mdx              (--bridge/-b, --help/-h table)
│   ├── faq.mdx                            (Q&A knowledge base)
│   └── troubleshooting.mdx                (README troubleshooting entries, expanded)
└── developer/                             ("Developer & Technical Documentation", position 3)
    ├── project-structure.mdx              (repo layout table)
    ├── architecture.mdx                   (data-flow diagram, REPL vs Bridge mode)
    ├── source-code-reference.mdx          (per-module reference: index/config/cli/bridge/remote)
    ├── acp-protocol.mdx                   (JSON-RPC methods & error codes implemented)
    └── contributing.mdx                   (build/dev loop, docs dev loop, conventions)
```

**End-User Documentation** covers user manuals (installation, configuration), how-to
guides/tutorials (CLI usage, Zed bridge usage), and an FAQ / knowledge base (FAQ +
troubleshooting).

**Developer & Technical Documentation** covers source code documentation
(source-code-reference, acp-protocol), project structure, and architecture design.
